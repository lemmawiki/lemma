# 속도계만 보고, 얼마나 멀리 왔는지 알 수 있을까?

창밖이 보이지 않는다. 속도계와 스톱워치만 있다. 속도는 변한다 — 어느 순간 30, 어느 순간 60, 어느 순간 0. **총 이동 거리는?** 직관: 속도가 "거의 일정한" 짧은 구간들로 나누고, 각 구간의 속도 × 시간을 더한다. 그게 리만 합이고, 그 합의 *극한* (구간이 줄어들 때) 이 <Term id="definite-integral">정적분</Term>이다. 이름이 다를 뿐, 같은 연산 — 변화의 누적.

<Lede>
  아래 위젯에서 <b>N</b> (직사각형 개수) 을 끌어보면 주황 합이 초록 정확값으로 다가간다. **속도**
  함수를 고르면 포물선 운동의 이야기가 반대 방향에서 복원된다 — 시간에 대한{" "}
  <span className="font-mono">g·t</span>의 적분이 떨어진 거리.
</Lede>

</Hook>

    함수에 적용되지 않는다.

<ArcRow n={1} title="넓이부터 — 일정한 속도의 누적">

가장 단순한 경우는 일정한 비율. <Formula>60 km/h</Formula>로 <Formula>2 h</Formula> 운전하면 <Formula>60 × 2 = 120 km</Formula>. 속도-시간 그래프에서는 직사각형 넓이 — 폭 × 높이. 곡선이 일정하지 않더라도 총 거리는 *속도 곡선 아래 넓이*. 이 모듈 전체가 그 단순한 경우를 *비일정한* 비율로 다시 짜는 일이다.

선형으로 변하는 비율 — 정지에서 떨어진 물체에 중력이 주는 <Formula>v(t) = g · t</Formula> — 의 경우 <Formula>[0, T]</Formula>에서 직선 아래 넓이는 *삼각형* — <Formula>(1/2) · T · (g · T) = (g · T²) / 2</Formula>. 그게 떨어진 거리. 새 물리도, 새 미적분도 없이 — *넓이*만으로.

</ArcRow>

<ArcRow n={2} title="리만 합 — 유한 장부질">

삼각형이 아닌 곡선은 기하만으로 넓이가 안 읽힌다. 대신 *근사*하자. 구간 <Formula>[a, b]</Formula>를 폭 <Formula>Δx = (b − a)/N</Formula>의 <Formula>N</Formula>개 띠로 자른다. 각 띠에서 표본점 하나 — 왼쪽 끝, 오른쪽 끝, 또는 중점, 선택 — 을 잡고 그 한 값을 띠의 "일정한" 높이로 쓴다. 띠 넓이를 모두 더한다. 결과가 <Term id="riemann-sum">리만 합</Term>:

<Formula>S_N = Σ f(x_i) · Δx</Formula>

위 위젯이 이걸 직접 보여준다. <Formula>f(x) = x²</Formula>를 <Formula>[0, 1]</Formula>에서 골라 N을 올려보자. <Formula>N = 1</Formula>에선 직사각형이 우스울 정도로 틀리고, <Formula>N = 8</Formula>에선 알아볼 만하고, <Formula>N = 80</Formula>에선 정확값 <Formula>1/3</Formula>과의 격차가 안 보인다. 다른 규칙 (왼쪽/오른쪽/중점) 은 같은 극한으로 다른 속도로 다가간다 — 왼/오른은 <Formula>O(1/N)</Formula>, 중점은 <Formula>O(1/N²)</Formula>. *그래서 모든 수치 적분기는 중점 또는 그 이상을 고른다.*

</ArcRow>

<ArcRow n={3} title="정적분 — 리만 합의 극한">

<Term id="definite-integral">정적분</Term>은 극한에 살아남는 것:

<Formula>{"∫_a^b f(x) dx = lim_(N → ∞) Σ_{i=1}^N f(x_i) · Δx"}</Formula>

*하나의 수* — 누적된 총량. 표기는 리만 이전부터 — <Formula>∫</Formula> 기호는 "sum"의 늘인 "S", <Formula>dx</Formula>는 극한이 줄여낸 <Formula>Δx</Formula>의 후손. "<Formula>{"∫_a^b f(x) dx"}</Formula>"를 "<Formula>a</Formula>에서 <Formula>b</Formula>까지 <Formula>f(x)</Formula> 곱하기 작은 <Formula>dx</Formula>들의 합"으로 읽는 건 정확한 직관 — 리만 이전의 뉴턴/라이프니츠 직관이기도 하다.

위젯의 "정확한 적분" 칸이 이 수다 — 닫힌 형식으로 계산 (쉬운 함수를 골랐기 때문). 대부분의 실제 함수에서는 극한이 수치적으로 취해지고, 컴퓨터가 적분을 실제로 어떻게 계산하는지의 답이 어떤 형태든 리만 합.

</ArcRow>

<ArcRow n={4} title="원시함수 — 미분의 역연산">

적분이 미분과 만난다. [미분 모듈](/ko/modules/derivatives)에서 <Formula>F(x) = x³</Formula>를 미분하면 <Formula>F'(x) = 3x²</Formula>. 거꾸로 읽으면 — <Formula>3x²</Formula>는 *원시함수를 갖는다* — 그것이 <Formula>x³</Formula>. <Formula>f</Formula>의 <Term id="antiderivative">원시함수</Term>는 <Formula>F'(x) = f(x)</Formula>를 만족하는 임의의 함수 <Formula>F</Formula>.

원시함수는 유일하지 않다 — <Formula>x³</Formula>와 <Formula>x³ + 5</Formula> 둘 다 <Formula>3x²</Formula>로 미분된다. 어떤 상수 <Formula>C</Formula>를 더해도 미분이 변하지 않으니, 교과서가 <Formula>F(x) + C</Formula>로 적는 이유. 한 <Formula>f</Formula>의 모든 원시함수의 집합은 한 매개변수 가족 — 서로 평행 이동된 함수들.

다항식의 경우 규칙은 기계적이다 — <Formula>x^n</Formula>의 미분이 <Formula>n · x^(n−1)</Formula>이니, <Formula>x^n</Formula>의 원시함수는 <Formula>x^(n+1) / (n+1)</Formula>. *공식의 장부를 거꾸로.* <Formula>sin x</Formula>의 원시함수는 <Formula>−cos x</Formula>, <Formula>e^x</Formula>의 원시함수는 <Formula>e^x</Formula> (출발한 자리). 이런 유한 목록과 연쇄법칙이 실무에서 만나는 대부분을 덮는다.

<ArcRow n={5} title="기본정리 — 넓이 = 미분의 역">

두 이야기 — *곡선 아래 넓이*와 *미분의 역연산* — 가 같은 이야기로 밝혀진다. <Term id="fundamental-theorem-of-calculus">미적분학의 기본정리</Term>가 다리:

<Formula>{"∫_a^b f(x) dx = F(b) − F(a)"}</Formula>

— *어떤* 원시함수 <Formula>F</Formula>에 대해서나. 리만 합의 무한 극한이 함수 평가 두 번과 빼기로 무너진다. 손으로든 계산기로든 *대부분의* 정적분은 이렇게 계산된다. 리만 극한은 적분이 *무엇인지*고, FTC는 *어떻게 계산하는지*다.

포물선 운동의 경우. 속도는 <Formula>v(t) = g · t</Formula>, 원시함수는 <Formula>(g · t²) / 2</Formula>. FTC에 따라 <Formula>0</Formula>에서 <Formula>T</Formula>까지 떨어진 거리는 <Formula>(g · T²)/2 − 0 = (g · T²) / 2</Formula> — § 1의 기하로 읽은 그 삼각형 넓이가 이제 원시함수에서 유도된다. 두 길, 한 수.

기본정리의 다른 절반 — <Formula>{"(d/dx) ∫_a^x f(t) dt = f(x)"}</Formula> — 이 양방향 연결을 단단하게 만든다. *미분이 적분을 되돌리고, 적분이 미분을 누적한다.* 둘은 한 교과서 챕터를 공유하는 별개 연산이 아니라, 덧셈과 뺄셈이 역인 것과 같은 의미에서 *역*이다.

</ArcRow>

  적분 자체. 원시함수가 계산 방법. 기본정리가 두 그림이 같은 그림임을 — 미분의 역으로서 — 말한다.

<Exercise number={1} tag="일정 속도 손계산" noCalculator>

차가 <Formula>60 km/h</Formula>로 2시간 일정하게 운전. 총 거리를 두 길로 계산하라 — (a) 속도-시간 그래프의 직사각형 넓이로, (b) 일정 함수 <Formula>v(t) = 60</Formula>의 원시함수를 구하고 FTC를 적용해.

<Solution>
  (a) 60 × 2 직사각형 넓이 = **120 km**. (b) 일정 <Formula>60</Formula>의 원시함수는{" "}
  <Formula>60 · t</Formula>; FTC: <Formula>60 · 2 − 60 · 0 = 120</Formula>. 같은 답, 다른 길. *일정
  속도가 보정점이다 — 모든 더 복잡한 적분은 "비율이 일정하지 않을 때의 넓이"이고, FTC가 그것을
  원시함수의 두 값을 빼는 일로 줄인다.*
</Solution>

</Exercise>

<Exercise number={2} tag="리만 합 손계산" noCalculator>

<Formula>f(x) = x²</Formula>를 <Formula>[0, 1]</Formula>에서 <Formula>N = 4</Formula>개 직사각형의 중점 리만 합으로 계산. (중점: <Formula>0.125, 0.375, 0.625, 0.875</Formula>; <Formula>Δx = 0.25</Formula>.) 정확값 <Formula>1/3 ≈ 0.3333</Formula>과 비교.

<Solution>
  중점 높이: <Formula>0.125² = 0.0156</Formula>, <Formula>0.375² = 0.1406</Formula>,{" "}
  <Formula>0.625² = 0.3906</Formula>, <Formula>0.875² = 0.7656</Formula>. 합:{" "}
  <Formula>0.0156 + 0.1406 + 0.3906 + 0.7656 = 1.3125</Formula>. <Formula>Δx = 0.25</Formula>를
  곱하면 <Formula>S_4 = 0.3281</Formula>. 정확값은 <Formula>1/3 ≈ 0.3333</Formula>; 오차{" "}
  <Formula>≈ 0.005</Formula>, 약 <Formula>1.5%</Formula>. 이미 <Formula>N = 4</Formula>에서
  쓸만하고, 두 배인 <Formula>N = 8</Formula>이면 오차가 <Formula>0.4%</Formula>로 떨어진다 (중점은{" "}
  <Formula>O(1/N²)</Formula>).
</Solution>

<Exercise number={3} tag="원시함수로 풀기">

FTC로 <Formula>{"∫_0^2 (3t² + 2t + 1) dt"}</Formula>를 계산. 위젯에서 <Formula>N = 100</Formula> 중점으로도 시도해 결과가 일치하는지 확인.

<Solution>
  원시함수: <Formula>F(t) = t³ + t² + t</Formula> (각 항을{" "}
  <Formula>x^n → x^(n+1)/(n+1)</Formula> 규칙으로 거꾸로). FTC:{" "}
  <Formula>F(2) − F(0) = (8 + 4 + 2) − 0 = 14</Formula>. 위젯이 이 함수를 직접 못 돌리지만 (프리셋
  세 개 고정), code-mode arc 5가 <Formula>N = 10000</Formula> 중점으로 돌리면 소수점 4자리에서
  일치. *FTC: 14, 정확, 함수 평가 두 번. 리만 합: 13.999..., 근사, 평가 10000번.* 둘 다 옳고, 한
  쪽이 훨씬 빠르다.
</Solution>

</Exercise>

<Exercise number={4} tag="속도에서 낙하 거리">

공이 정지에서 떨어진다. 중력이 일정 가속도 <Formula>g = 9.8 m/s²</Formula>를 주니, 시간 <Formula>t</Formula> 후 속도는 <Formula>v(t) = g·t</Formula>. <Formula>t = 3 s</Formula>까지 얼마나 떨어졌나?

<Solution>
  거리는 <Formula>{"∫_0^3 g·t dt = g · t²/2 |_0^3 = (9.8)(9)/2 = 44.1 m"}</Formula>. 동등하게, 밑변{" "}
  <Formula>3</Formula>, 높이 <Formula>v(3) = 29.4 m/s</Formula>인 삼각형의 넓이는{" "}
  <Formula>(1/2)(3)(29.4) = 44.1 m</Formula>. 두 길, 같은 수. *이게 포물선 운동 페이지의{" "}
  <Formula>y(t) = (1/2) g t²</Formula> 공식 — 한 번은 운동학에서, 한 번은 여기서 "v를 적분해 y를
  얻는다"로 유도.*
</Solution>
