# What is "the same curve"?

Drawing the parabola <Formula>y = x²</Formula> on a board, you'd say _this is a curve_. Below you'll see the same picture drawn three different ways — same paint, different motions. Which one is **the** curve? Pick an answer in your head before pressing play. Then play.

<ArcRow n={1} title="γ is a function, not a shape">

A <Term id="parametrized-curve">parametrized curve</Term> is a function <Formula>γ : [0, 1] → ℝ²</Formula>. The three γ above are three _different_ functions. They happen to share an <Term id="image-of-curve">image</Term> — the set <Formula>{"{γ(t) : t ∈ [0, 1]}"}</Formula>, which is the parabola <Formula>y = x²</Formula> on <Formula>x ∈ [−1, 1]</Formula> — but the image is a _set of points_, stripped of order, speed, and visit count. The function γ is none of those things. Two parametrized curves are equal when <Formula>γ₁(t) = γ₂(t)</Formula> for every t — the same equality you'd use for any other function. By _that_ test, the three γ above are three different curves that happen to leave the same shadow. But functional equality is only the strictest of three layers; the next step locates where the line should actually be drawn.



</ArcRow>

<ArcRow n={2} title="Same image, but is it the same curve?">

Step 1's strict test separates γ₁ from γ₂ even though they paint the same picture in the same direction at different paces. That feels too strict — geometrically they ought to be the same. Differential geometry agrees: it identifies γ with <Formula>γ ∘ φ</Formula> whenever <Term id="reparametrization">φ</Term> is a monotone bijection <Formula>[0, 1] → [0, 1]</Formula>. With <Formula>φ(u) = ((2u−1)³ + 1) / 2</Formula> (monotone since <Formula>dφ/du = 3(2u−1)² ≥ 0</Formula>), one checks <Formula>γ₁ ∘ φ = γ₂</Formula> exactly: γ₁ and γ₂ are the same curve at different paces. γ₃ has no such partner — it visits each interior image point twice, and no monotone φ can take a once-visit schedule to a twice-visit one. So between strict equality (too tight) and image equality (too loose) sits <Term id="reparametrization">reparametrization</Term> equivalence — and that is what most working mathematicians mean by "the same curve."



</ArcRow>

<ArcRow n={3} title="Where this shows up — same γ, two pillars">

**The same map γ(t) can be a designer's stroke, a projectile's path, or a clock's motion.** Once "curve" stops meaning *shape* and starts meaning *function-of-time*, the same machinery splits cleanly across pillars.

<pre className="my-2 overflow-x-auto rounded-md bg-rule-soft p-3 font-mono text-[13px] leading-[1.55] text-ink">{`graphics : γ is design — a stroke a user wants to see.
physics  : γ is motion — a body's position at time t.`}</pre>

[**Bezier curves**](/en/graphics/bezier-curves) — the curve a designer pulls is literally a parametrized γ : [0, 1] → ℝ². Dragging a handle changes γ; the picture changes because γ has changed. The whole de Casteljau evaluator is "given t and the control points, compute γ(t)." Same tool, used as drawing.

[**Projectile motion**](/en/physics/projectile-motion) — γ(t) = (v₀ cos θ · t, v₀ sin θ · t − ½ g t²) is the position at time t. Here time is the parameter and the *image* is the trajectory through space. Same tool, used as motion.

[**The pendulum clock**](/en/physics/pendulum-clock) — the bob's tip traces a γ over time as θ(t) swings. The image is an arc; the function is the schedule that says when the bob is where. The escapement only works because that schedule is *periodic in t* — a property of γ, not of its image.

**The split arc 2 made — image vs. parametrization — is exactly what lets a designer's drag and a physicist's pendulum both be γ. The image is what each pillar shows; γ is what each pillar manipulates.**

</ArcRow>

<Pin>
**Three layers of "same":**
- **same image** — the picture: γ₁ ~ γ₂ ~ γ₃
- **same up to reparametrization** — the geometric curve: γ₁ ~ γ₂; γ₃ stands alone
- **equal as functions** — the parametrized curve: all three differ

The picture is the weakest layer; the function is the strictest. **Geometry lives in between.** School math conflates all three under the single word _curve_; that is why "is this the same curve?" has no clean answer — the word was quietly carrying three different objects. The widget you played with above is one diagram showing all three at once.

</Pin>

<Exercise number={1} tag="read the widget · trail density">

Play γ₂ and look at the trail. Where are the points _most_ tightly packed — near the origin, or near the endpoints? Why? (Trail spacing is constant in time, not in arc length.)

<Solution>
  Densest *at the origin*. With <Formula>x(u) = (2u−1)³</Formula>, the speed{" "}
  <Formula>dx/du = 6(2u−1)²</Formula> vanishes at <Formula>u = 1/2</Formula> — γ₂ crawls through the
  origin and sprints through the endpoints. Equal time steps spend most of their time near the slow
  point, so trail dots pile up there. The image is the same parabola; only the schedule differs.
</Solution>

</Exercise>

<Exercise number={2} tag="construct · same image, different schedule" noCalculator>

Write a γ : [0, 1] → ℝ² whose image is exactly <Formula>y = x²</Formula> on <Formula>x ∈ [−1, 1]</Formula>, but which traces that parabola _three times_ as t goes 0 → 1. Give a closed-form expression. (Hint: the round trip γ₃ traces it twice using cosine.)

<Solution>
  One choice: <Formula>γ(t) = (cos(3πt), cos²(3πt))</Formula>. As t runs 0 → 1, the argument{" "}
  <Formula>3πt</Formula> sweeps <Formula>[0, 3π]</Formula>; cosine completes one and a half full
  periods, traversing <Formula>[−1, 1]</Formula> three times. Image: the parabola{" "}
  <Formula>y = x²</Formula> on <Formula>[−1, 1]</Formula>. Same shadow, triple-pace schedule. The
  image cannot tell this trip apart from γ₁'s single one.
</Solution>

</Exercise>

<Exercise number={3} tag="the evil one · 'extensional equality'">

A junior says: "Two functions with the same outputs are the same function — that's extensional equality. Three γ producing the same image must be the same curve." Find the bug in one sentence. Then state the correct equality test for parametrized curves.

<Solution>
  The bug: extensional equality of γ compares <Formula>γ(t)</Formula> at every t, not{" "}
  <Formula>{"{γ(t) : t}"}</Formula> as a set; the junior swapped the function for its image. Correct
  test: <Formula>γ₁ = γ₂ ⇔ γ₁(t) = γ₂(t) for all t</Formula>. Two γ with the same image are equal
  iff the schedule also matches, point by point.
</Solution>

</Exercise>
