# 모델이 틀렸다. 어떤 손잡이를 어느 방향으로 얼마나 움직여야 할까?

매개변수 <Formula>w</Formula> 하나, 모델이 얼마나 틀렸는지를 재는 <Term id="loss-function">손실</Term> <Formula>L(w)</Formula> 가 있다. *어느 방향*으로 <Formula>w</Formula>를 움직여야 <Formula>L</Formula>이 작아질까? _얼마나_? 답: 방향은 <Term id="derivative">미분</Term>이 알려주고, 거리는 <Term id="learning-rate">학습률</Term>이라 부르는 한 숫자가 결정한다. 둘을 합친 것이 수치 수학에서 가장 단순하고 가장 오래된, 그리고 가장 많이 쓰이는 레시피 중 하나 — 모든 현대 ML 시스템의 엔진.

<Lede>
  아래 위젯에서 <Formula>w₀</Formula> 출발점과 <Formula>η</Formula>를 끌고 **한 스텝**을 눌러보자.
  세 프리셋: *작음*은 기어 내려가고, *적당*은 두어 스텝 만에 최솟값에 닿고, *너무 큼*은 진동하며
  발산한다. "적당"과 "발산" 사이의 경계가 이 방법의 안정성 이론 *전부* — 한 화면 안에.
</Lede>

</Hook>

<ArcRow n={1} title="틀림을 숫자로">

가장 단순한 모델: 매개변수 하나 <Formula>w</Formula>, 입력 하나 <Formula>x</Formula>, 예측 <Formula>ŷ = w · x</Formula>. 우리는 참값 <Formula>y</Formula>에 대해 <Formula>ŷ ≈ y</Formula>를 원한다. "가까움"엔 한 숫자가 필요한데, 표준 선택은 제곱 오차 <Formula>L(w) = (w·x − y)²</Formula> — 예측이 정확하면 0, 멀면 크고, (중요하게) 모든 곳에서 _매끄럽다_ 미분이 가능하도록.

위젯은 <Formula>(x, y) = (2, 6)</Formula>로 고정 — 손실은 <Formula>L(w) = (2w − 6)² = 4(w − 3)²</Formula>, 최솟값은 <Formula>w\_ = 3</Formula>인 포물선. 문제 전체는 한 줄로 줄어든다 — 임의의 출발점 <Formula>w₀</Formula>에서 <Formula>3</Formula>까지 어떻게 갈 것인가.

</ArcRow>

<ArcRow n={2} title="어느 방향? — 미분">

[미분 모듈](/ko/modules/derivatives)이 한 줄로 답한다: <Formula>L'(w) = 2x(wx − y)</Formula> — 현재 <Formula>w</Formula>에서의 손실 곡선의 기울기. 만약 <Formula>L'(w) &gt; 0</Formula>이면 <Formula>w</Formula>가 증가할수록 손실이 _상승_ — <Formula>w</Formula>를 _줄여야_ 한다. <Formula>L'(w) &lt; 0</Formula>이면 반대. 하강 방향은 항상 <Formula>−L'(w)</Formula>.

위젯에서 현재 점의 초록 화살표가 그 방향. <Formula>w</Formula>가 최솟값에 가까워지면 화살표가 줄고, 정확히 <Formula>w\_</Formula>에서 길이가 0 — 최솟값에서 기울기가 정확히 0이라 남는 신호가 없다.

<CodeBlock slug="gradientDescent" arc="arc2" />

</ArcRow>

<ArcRow n={3} title="얼마나? — 학습률">

방향은 정해졌다. 크기는 아직 아니다. 기울기에 양수 <Formula>η</Formula> (<Term id="learning-rate">학습률</Term> · <Dialect concept="learning-rate" />) 를 곱해 빼준다:

<pre className="my-2 overflow-x-auto rounded-md bg-rule-soft p-3 font-mono text-[13px] leading-[1.55] text-ink">{`w  ←  w  −  η · L'(w)`}</pre>

<Formula>η</Formula>가 다르면 궤적이 다르다. 위젯에서 <Formula>η = 0.04</Formula>로 보면, 매
스텝마다 <Formula>w_</Formula>까지의 거리가 일정 비율 (여기선 약 0.68) 줄어드는 기하 수렴 — 느리지만
안정. <Formula>η = 0.12</Formula>는 뉴턴-최적 <Formula>η = 1/L''(w_) = 1/8 = 0.125</Formula> 근처 —
두세 스텝이면 끝. <Formula>η = 0.27</Formula>은 경계 너머. 무슨 일이 일어나는지 다음 절에서 본다.

</ArcRow>

<ArcRow n={4} title="반복 — 하강 루프">

방법 전체는 스텝 → 스텝 → 스텝 — 기울기가 충분히 작아지거나 (최솟값 근처) 한도가 끝날 때까지:

<pre className="my-2 overflow-x-auto rounded-md bg-rule-soft p-3 font-mono text-[13px] leading-[1.55] text-ink">{`for _ in range(steps):
    w = w − η · L'(w)`}</pre>

코드 다섯 줄. 그게 경사하강법이다. 위젯의 *자동*이 정확히 이 루프를 돌린다. <Formula>η = 0.12</Formula>로 누르면 <Formula>w</Formula>가 <Formula>3</Formula>으로 수렴. <Formula>η = 0.27</Formula>로 누르면 갱신값이 화면 밖으로 날아간다.

<CodeBlock slug="gradientDescent" arc="arc4" />

</ArcRow>

<ArcRow n={5} title="발산 — η가 너무 크면">

우리의 이차 손실에서 <Formula>u = w − w_</Formula>로 두면 갱신이 <Formula>u ← (1 − η · L''(w_)) · u</Formula> — *최솟값까지의 거리*가 매 스텝 <Formula>r = 1 − η · L''(w_)</Formula>로 곱해진다. 안정 수렴은 <Formula>|r| &lt; 1</Formula>, 즉 <Formula>0 &lt; η &lt; 2/L''(w_)</Formula>. 우리 손실의 <Formula>L''(w) = 8</Formula>이라 <Formula>η &lt; 0.25</Formula>는 수렴, <Formula>η &gt; 0.25</Formula>는 발산.

<Formula>0.25</Formula> 위에서는 매 스텝이 최솟값을 *교정한 것보다 더 멀리* 지나친다.
<Formula>u</Formula>의 부호가 번갈아가며 (최솟값의 양쪽에 번갈아 떨어진다) 크기가 자란다. 위젯이
그림을 보여준다 — <Formula>η = 0.27</Formula>에서 갱신값이 핑퐁하며 부풀어 오른다. 실제 ML의 해법은
단순한 시행착오 하나가 아니라, 국소 곡률을 *측정*하는 것 (2차 방법, Adam 같은 적응형) + 최적해에
가까워질수록 <Formula>η</Formula>를 *줄이는* 스케줄.

</ArcRow>

<ArcRow n={6} title="왜 ML 곳곳에 이 식이 있는가">

실제 ML은 매개변수가 하나가 아니라 수백만~수십억. 예제도 하나가 아니라 수백만. 손실은 깔끔한 포물선이 아니라 안장점과 평탄지가 가득한 고차원 비볼록 풍경. 어느 것도 레시피 자체를 바꾸지는 못한다 — 각 기호가 가리키는 *것*만 바뀐다.

<ul className="m-0 mb-3 list-none space-y-1.5 pl-0 [&_li]:border-l-[3px] [&_li]:border-rule [&_li]:pl-3.5 [&_b]:font-semibold [&_b]:text-ink">
  <li>
    **매개변수 하나 → 벡터.** <Formula>w</Formula>가 매개변수 <Term id="vector">벡터</Term>{" "}
    <Formula>θ</Formula>로, 기울기는 편미분의 벡터로. 스칼라 갱신 대신 벡터 뺄셈 — [벡터
    모듈](/ko/modules/vectors)이 깔아둔 두 연산 (덧셈, 스칼라배) 그대로.
  </li>
  <li>
    **예제 하나 → 미니배치.** 매 스텝마다 작은 무작위 샘플에 대해 손실을 합 (또는 평균). 기울기가
    *잡음 있음* — 이게 <Formula>SGD</Formula>. 잡음은 버그가 아니라 특징 — 나쁜 국소 기하를 탈출하는
    데 도움.
  </li>
  <li>
    **깔끔한 포물선 → 실제 손실 표면.** 깊은 신경망의 교차 엔트로피는 비볼록. 하나의 최적해는 없고,
    같은 하강 루프로 찾아지는 "충분히 좋은" 국소 최솟값으로 만족.
  </li>
  <li>
    **손-유도 기울기 → autograd.** 연쇄 법칙은 기계적이다. 현대 ML 프레임워크는 forward 동안 계산
    그래프를 만들고 그것을 거꾸로 걸어 자동으로 <Formula>∇L</Formula>을 만든다. 우리가 토이에서
    손으로 계산한 미분과 *같은 연산*, 규모만 다르다.
  </li>
</ul>

[자신 있게 틀리기](/ko/ml/confident-wrong)로 가는 다리: 여기 제곱 손실은 워밍업. 거기 교차 엔트로피 손실도 같은 "기울기 계산 → 한 스텝" 기하를 가진다 — 단지 <Formula>(ŷ − y)²</Formula> 대신 <Formula>−log p_true</Formula>일 뿐. 같은 하강 알고리즘, 다른 풍경.

<Exercise number={1} tag="손실 계산" noCalculator>

<Formula>x = 2</Formula>, <Formula>y = 6</Formula>, <Formula>w = 1</Formula>일 때 예측
<Formula>ŷ</Formula>와 손실 <Formula>L(w) = (ŷ − y)²</Formula>를 계산하라.

<Solution>
  <Formula>ŷ = 1 · 2 = 2</Formula>. 차이는 <Formula>(2 − 6) = −4</Formula>. 손실{" "}
  <Formula>(−4)² = 16</Formula>. 위젯에서 <Formula>w = 1</Formula>로 두고 확인.
</Solution>

</Exercise>

<Exercise number={2} tag="기울기 손계산" noCalculator>

[미분 모듈](/ko/modules/derivatives)의 할선-접선 레시피로 <Formula>L'(w) = 2x(wx − y)</Formula>를 처음부터 유도하라. 그 다음 <Formula>(x, y) = (2, 6)</Formula>, <Formula>w = 1</Formula>에서 값을 계산.

<Solution>
  <Formula>L(w + h) − L(w) = ((w + h)x − y)² − (wx − y)²</Formula>. 제곱을 펴고 빼면{" "}
  <Formula>2(wx − y)·xh + x²h²</Formula>. <Formula>h</Formula>로 나누면{" "}
  <Formula>2x(wx − y) + x²h</Formula>. <Formula>h → 0</Formula>:{" "}
  <Formula>L'(w) = 2x(wx − y)</Formula>. <Formula>w = 1</Formula>에서:{" "}
  <Formula>2 · 2 · (2 − 6) = 4 · (−4) = −16</Formula>. 음수 — <Formula>w</Formula>가 늘면 손실이
  감소, 그래서 하강이 <Formula>w</Formula>를 *위쪽*으로 민다. ✓
</Solution>

</Exercise>

<Exercise number={3} tag="한 스텝 손계산" noCalculator>

<Formula>w = 1</Formula>에서 출발, <Formula>η = 0.12</Formula>, 기울기 <Formula>−16</Formula>일 때
한 스텝 뒤 <Formula>w</Formula>는? 두 스텝 뒤는? 최적해 근처에 빨리 닿을 것이다.

<Solution>
  스텝 1: <Formula>w = 1 − 0.12 · (−16) = 1 + 1.92 = 2.92</Formula>. 기울기 다시 계산:{" "}
  <Formula>2 · 2 · (5.84 − 6) = −0.64</Formula>. 스텝 2:{" "}
  <Formula>w = 2.92 − 0.12 · (−0.64) = 2.92 + 0.077 ≈ 3.00</Formula>. 두 스텝 만에 최적해.{" "}
  <Formula>η = 0.12</Formula>가 뉴턴-최적 <Formula>1/L''(w_) = 1/8 = 0.125</Formula>에 가까워서.
</Solution>

</Exercise>

<Exercise number={4} tag="왜 큰 η가 폭발하는가">

우리의 이차 손실에서 변위 <Formula>u = w − w\_</Formula>의 갱신은 <Formula>u ← (1 − 8η) · u</Formula>. <Formula>η = 0.27</Formula>에서 <Formula>|u_n| → ∞</Formula>이 <Formula>n</Formula>에 대해 지수적으로 발산함을 보여라. 어느 <Formula>η</Formula>에서 갱신이 경계에 머무르는가 (진폭 일정, 자라지도 줄지도 않음)? 그 <Formula>η</Formula>가 특별한 이유는?

<Solution>
  <Formula>r = 1 − 8 · 0.27 = 1 − 2.16 = −1.16</Formula>; <Formula>|r| = 1.16 &gt; 1</Formula>.
  그러므로 <Formula>|u_n| = |r|^n · |u_0| = 1.16^n · |u_0|</Formula> — 지수적 성장. 경계는{" "}
  <Formula>|r| = 1</Formula> — 즉 <Formula>r = 1</Formula> (갱신 없음, <Formula>η = 0</Formula>
  에서만) 또는 <Formula>r = −1</Formula> (<Formula>w_</Formula> 주위 완전 진동), 후자는 **η = 0.25 =
  2/L''(w*)**에 해당. 이 임계 <Formula>η</Formula>에서 매 스텝 부호가 뒤집히지만 줄지 않음 —
  알고리즘이 수렴과 발산의 경계에 앉아 있다. 실제 ML 훈련은 이 경계보다 *훨씬* 아래에 있다 (보통
  10배 이상), 특히 곡률을 모르거나 매개변수마다 다를 때.
</Solution>

</Exercise>
