# 사라진 교점들은 어디로 갔을까?

화면 위 두 이차곡선을 끌어보라. 어떤 때는 교점이 **네 개**, 어떤 때는 **둘**, **하나**, **없음**. 충돌을 계산하는 물리 엔진, 패스를 잘라내는 벡터 도구, 외곽선을 다듬는 폰트 래스터라이저, 픽을 처리하는 레이 트레이서 — 모두 "몇 개?"라는 물음에 _흔들리지 않는_ 답이 필요하다. 화면 위의 수는 깜빡거린다. 그 아래 수학은 깜빡이지 않는다.

<Lede>
  [베주 정리](/ko/modules/bezout)는 두 이차곡선의 교점이 늘 **4개**라고 말한다 — 올바른 수 체계,
  올바른 평면, 올바른 중복도로 셀 때. 아래 위젯은 베주 모듈에 실린 위젯과 *같은 것*이다. 다만 여기서
  답하는 질문이 다르다: *그 4개 중에서 화면이 *실제로* 보여주는 건 어느 것인가?*
</Lede>

</Hook>

<ArcRow n={1} title="화면이 보는 것 vs 거기 있는 것">

2D 평면 위의 두 실수 곡선. *눈에 보이는 셈*이란 _같은 픽셀에서 두 곡선이 만나는 점의 개수_ — 좌표가 실수이고 양쪽 모두 유한한 경우다. 래스터 샘플링으로 답을 내는 단순 충돌 검출기, 근사 근-찾기로 답을 내는 CAD 엔진이 모두 이 셈을 내놓는다. 곡선이 지금 어디쯤 떠 있느냐에 따라 달라지고, 작은 섭동에도 쉽게 흔들린다.

위젯에서 확인해 보자. *일반* 프리셋은 실수 교점 4개 — 보이는 셈과 베주의 셈이 맞아떨어진다. *접* 프리셋에서는 3개로 보이지만, *중복도 표시*를 켜면 그중 하나에 $×2$가 붙는다. *분리* 프리셋에서는 0개로 보이지만, *복소 보기*를 켜면 옆 패널에 빈 동그라미 네 개가 떠오른다. 사라진 교점들은 잃어버린 것이 아니다. 단지 실-아핀 평면을 떠났을 뿐.

</ArcRow>

<ArcRow n={2} title="세 가지 숨는 곳">

베주의 보정([베주 모듈](/ko/modules/bezout)에서 자세히 다룬다)은 사라진 교점이 어디로 갔는지 정확히 짚어준다. 보정은 셋, 들여다볼 자리도 셋이다.

- **접점.** 두 교점이 한 기하점으로 포개진 경우 — 대수는 여전히 둘이라고 말하지만 렌더러 눈에는 한 점만 보인다. 이걸 장부에 적는 방식이 <Term id="intersection-multiplicity">교차 중복도</Term>다. 접 교점은 2로 센다.
- **보이는 평면 바깥.** 교점의 좌표가 복소수인 경우 — 대수의 해로는 분명히 존재하지만 실수 $(x, y)$ 격자 위에는 찍을 자리가 없다. 떨어져 있는 두 단위원조차 $(3/2, ± i \sqrt{5} / 2)$에서 만난다. 화면에는 이 점을 놓을 곳이 없다.
- **무한 멀리.** 어떤 교점은 무한원선 위에 자리 잡는다. 서로 다른 두 원은 어디에 놓여 있든 늘 두 _원형점_ $I = [1 : i : 0]$과 $J = [1 : −i : 0]$을 공유한다 — 베주의 4개 중 2개는 이미 화면 밖, 복소수, 좋든 싫든 장부에 적힌 셈이다. 이 보정의 무대가 <Term id="projective-plane">사영평면</Term>이다.

</ArcRow>

<ArcRow n={3} title="그래픽 엔진이 *흔들리지 않는 수*를 챙기는 이유">

"보이는 셈"만 좇는 엔진은 깨지기 쉽다. 거의 접하는 두 원의 보이는 셈은 서브픽셀 떨림에 따라 0이 되었다 1이 되었다 2가 된다. 한 프레임에 "교점 없음", 다음 프레임에 "교점 둘"이라 보고하는 렌더러는 결국 수치 잡음을 물리로 둔갑시키고 있는 것이다. 답은 _대수적_ 셈 — 베주가 보장해 주는 바로 그 셈 — 을 먼저 구하고, 각 근을 *분류*하는 데 있다.

실제 절차는 이렇다. y-종결식(x에 대한 4차)을 푼다. 중복도까지 포함해 복소근 4개를 모두 찾는다. 각각을 _실-아핀_(보임), _복소_(평면 밖), _무한원_(점근) 중 하나로 분류한다. 첫 번째 부류의 개수가 곧 보이는 셈이다. 접점은 "우연히 매우 가까운 두 근"이 아니라 *중복근*으로 표시된다. *중복도 2인 실수 근 하나*와 *서로 ε 안에 있는 서로 다른 실수 근 둘*은 화면 위에서는 똑같아 보이지만, _기하적으로는 전혀 다른 상황_이다 — 대수 분류가 이 둘을 갈라놓는다.

</ArcRow>

<ArcRow n={4} title="엔진이 실제로 계산하는 것">

파이프라인은 짧다. (1) 두 이차곡선 식에서 <Term id="resultant">한 변수를 소거</Term>해 다른 변수만 남은 단일 다항식을 얻는다 — 둘 다 차수 2이면 4차식이 나온다. (2) 이 다항식의 근을 수치적으로 구한다(Durand–Kerner 같은 방법으로). (3) 대응하는 y 좌표를 선형 역대입으로 복원한다. (4) 각 교점에 실-아핀, 복소, 무한 중 하나의 꼬리표를 단다. 위쪽 위젯은 슬라이더를 움직일 때마다 (1)–(4)를 처음부터 다시 돌린다.

<ArcRow n={5} title="그래픽 스택 안의 자리">

같은 _베주 + 분류_ 로직이 모양만 바꿔서 곳곳에서 돌아간다.

- **충돌 검출.** 두 볼록 도형(원·타원·캡슐)이 몇 번 만나는가? 베주가 상한을 잡아 주고, 접 + 중복도가 스치는 접촉을 잡아내며, "실수 0 + 복소 4"는 곡선이 깔끔히 떨어져 있다는 뜻이 된다.
- **벡터 클리핑.** Illustrator나 Figma에서 두 경로의 교차를 구할 때, 도구는 3차 베지에 곡선 사이의 대수 교점을 계산한다 — 차수 3 × 차수 3 = 최대 9개를 보임 / 구간 밖 / 접으로 나눈다.
- **레이-곡선 픽.** 직선(차수 1)과 이차곡선(차수 2)은 대수적으로 2개의 점에서 만난다. 마우스 클릭은 레이를 따라 처음 만나는 실-아핀 교점을 고른다. 둘 다 복소수면 클릭은 그대로 빗나간다.
- **폰트 외곽선 래스터화.** 수평 스캔라인(차수 1)이 글리프 외곽선(최대 3차의 베지에)을 d·1 = d개의 점에서 가로지른다. 중복도까지 함께 세면 짝-홀 채움 규칙이 *덤으로* 따라온다.

겉 이야기는 어디서나 "세고 분류한다"이고, 그 밑의 이야기는 베주 정리와 근 분류기다. 입는 옷은 달라지지만, *셈의 규약*은 변하지 않는다.

</ArcRow>

<Exercise number={1} tag="직선과 원" noCalculator>

직선 $y = mx + c$는 단위원 $x² + y² = 1$과 베주 셈으로 몇 점에서 만나는가? 어떤 $(m, c)$의 범위에서 그 점들이 모두 보이고(실-아핀), 어떤 범위에서 일부가 복소수로 숨는가? 각 경우의 예를 하나씩 그려 보라.

<Solution>
  베주: 1·2 = **2**. 대입하면 $(1+m²)x² + 2mc·x + (c²−1) = 0$, 판별식은{" "}
  $4(m² + 1 − c²)$. $c² &lt; m² + 1$이면 **실-아핀 둘**(직선이
  원반을 가로지른다). $c² = m² + 1$이면 **접(중복도 2)**.{" "}
  $c² &gt; m² + 1$이면 **복소 둘**(실평면에서는 원과 만나지 않는다). 세 경우 모두
  베주의 셈은 똑같이 2다.
</Solution>

</Exercise>

<Exercise number={2} tag="두 원은 무한원을 공유한다">

서로 다른 두 원을 잡자. 베주는 4번 만난다고 한다. 그런데 화면 위 그림에서는 실-아핀 교점이 늘 2개를 넘지 않는다. 나머지 2개는 _언제나_ 어디에 있는가? (힌트: 원의 <Term id="homogenize">동차화</Term> 식을 무한원선 $Z = 0$에 넣으면 최고차 항만 남는다.)

<Solution>
  $x² + y² + ax + by + c = 0$을 동차화하면{" "}
  $X² + Y² + aXZ + bYZ + cZ² = 0$. $Z = 0$에서는{" "}
  $X² + Y² = 0$이 되어 $[X : Y : 0] = [1 : ± i : 0]$ —{" "}
  $a, b, c$와는 아무 상관이 없다. 즉 *모든* 원은 무한원에서{" "}
  $I = [1 : i : 0]$과 $J = [1 : −i : 0]$을 지난다. 서로 다른 두
  원은 이 둘을 함께 공유한다. 베주의 4 = (실-아핀 ≤ 2) + (무한원의 복소 2). 실-아핀의 수가 2
  아래로 떨어질 때마다, 그 빈자리는 중복도나 복소-아핀 교점이 메워 준다.
</Solution>

</Exercise>

<Exercise number={3} tag="접 vs 거의-빗나감" noCalculator>

엔진에 매개변수가 둘 있다고 하자. 각도 $θ$와 허용오차 $ε$. $θ$가 접 배치를 통과할 때, 보이는 셈은 2 → 1 → 0 → 1 → 2로 변한다. _접_ 순간은 하나의 배치, 그 양옆의 _거의-접_ 순간은 또 다른 배치다. 그 차이를 대수적으로 진술하고(중복도 2인 근 하나 vs ε 안에 놓인 서로 다른 두 근), 견고한 엔진이 단순한 $ε$-거리만으로 중복 제거를 해서는 안 되는 이유를 설명하라.

<Solution>
  접 순간에는 종결식이 *중근* $x = a$를 갖는다 — 다항식이{" "}
  $(x − a)² · q(x)$로 인수분해된다. 거의-접은 서로 가까운 *단근* 둘{" "}
  $a ± δ$이다. $ε &gt; δ$인 허용오차 중복 제거기 눈에는 둘 다 "한
  점"으로 보인다. 하지만 접 쪽은 물리가 다르고(가로지르지 않는다 — 곡선이 스쳐 지나갔다 떨어진다),
  섭동에 대한 거동도 다르다(한 방향으로는 흔들림에 안정적, 반대 방향으로는 불안정). 견고한 엔진이라면
  가능한 한 판별식을 정확히 계산하거나, 구간 산술로 중복도 판정의 경계를 잡는다 — 거리 하나에만
  기대지 않는다.
</Solution>

</Exercise>

<Exercise number={4} tag="맞지 않는 삼차">

수평 직선 $y = 0$과 3차 곡선 $y = x³ + 1$의 교점을 구해 보자. 베주 예측은 1·3 = 3. 실-아핀은 몇 개이고, 나머지는 어디에 있는가? (보이는 점에는 $i$가 필요 없고, 나머지에는 각각 하나씩 들어간다.)

<Solution>
  $x³ + 1 = 0$의 해는 −1의 세제곱근이다: $x = −1$,{" "}
  $x = e^(iπ/3) = \tfrac{1}{2} + i\sqrt{3}/2$, $x = e^(−iπ/3) = \tfrac{1}{2} − i\sqrt{3}/2$.
  **실-아핀 교점은 하나** — $(−1, 0)$이고, 그래프에 찍히는 유일한 점이다. **복소
  교점 둘**은 켤레쌍으로, 실평면 바깥에 있다. 베주의 셈은 3, 그림의 셈은 1, 사라진 몫은 복소 2개다.
</Solution>

</Exercise>
